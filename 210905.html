서버와 클라이언트(form)

form은 사용자가 입력한 데이터(입력값)를 서버로 보내기 위해 사용하는 태그 (서버란,정보를 제공하는 호스트)

사용자(클라이언트)가 요청을 하면, 서버는 그에 대한 응답으로 정보를 제공
ex)사용자가 검색창에 주소를 입력- 호스트에게 요청-호스트는 응답으로 검색창에대한 결과페이지를 보냄
사용자(클라이언트)가 어떤 요청을 보내는가에 따라 응답은 달라질수 있다, 로그인 양식은 세개의 입력 요소로 구성되어 있으며,
세개의 입력 데이터는 form을 통해 서버로 전송된다

from은 입력 요소들을 감싸며, 입력 값을 서버 측으로 제출(submit)할 수 있다.

결과값 ↓

서버 측에서 데이터를 처리한 결과를 클라이언트에게 보내준다(응답)

form의 속성들

대표적 속성 두가지
action : 입력값을 전송할 서버의 url (url은 서버 프로그램의 주소나 파일명등을 의미함)
method : 입력 값을 전송할때의 방식(get&post)

ABC.php 라는 서버 프로그램으로 입력값을 전송하여 요청할 것이다. post방식으로 전송할 것이다.

get : 서버에 요청을 보내어 응답을 받아낸다
서버로부터 정보를 '가져오겠다'는 성격의 요청이다.

post : 서버에 요청을 보내어 작업을 수행한다 서버에 있는 데이터를
추가/수정/삭제 한 후에 응답을 받아낸다. 서버의 정보를 '조작하겠다'는 성격의 요청

내용 정리

1.form은 사용자가 입력 요소에 입력한 데이터를 서버로 전송해준다.
2.서버란 데이터를 제공하는 호스트이다.
3.클라이언트란 데이터를 제공받아 이용하는 사용자 이다.
4.form은 입력 요소(input,select,textarea 등등)를 감싸고, 데이터를 제출(submit)한다.
5.form의action은 서버 측 주소를 지정하는 속성이다.
6.form의 method는 데이터 전송 방식을 지정하는 속성이다.
